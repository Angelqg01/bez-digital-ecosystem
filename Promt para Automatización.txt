import subprocess
import json

# Lista simulada de leads extraídos de LinkedIn
leads = [
    {"nombre": "Carlos", "perfil": "Especialista en DAOs y protocolos DeFi. Busco gobernanza real."},
    {"nombre": "Elena", "perfil": "Inversionista institucional interesada en activos Web3 con voto."}
]

def procesar_con_gemini(lead_info):
    # Prompt optimizado para LinkedIn (Tono profesional)
    prompt = f"""
    Analiza este perfil de LinkedIn: "{lead_info['perfil']}".
    Genera un mensaje de conexión para la plataforma BeZhas. 
    Explica por qué cambiar sus tokens por Gobernanza de BeZhas le da poder real.
    Sé breve, profesional y evita palabras de 'vendedor de humo'.
    """
    
    # Llamada a Gemini CLI
    process = subprocess.Popen(['gemini', prompt], stdout=subprocess.PIPE, stderr=subprocess.PIPE, text=True)
    mensaje, _ = process.communicate()
    return mensaje

for lead in leads:
    propuesta = procesar_con_gemini(lead)
    print(f"--- Mensaje para {lead['nombre']} ---")
    print(propuesta)

import subprocess

def generar_propuesta_economica(tipo_lead, volumen_estimado):
    """
    Usa Gemini CLI para redactar una comparativa de ahorro personalizada.
    """
    contexto = f"""
    Genera una propuesta de venta para un lead tipo '{tipo_lead}'.
    Volumen de transacciones estimado: {volumen_estimado}.
    
    Incluye:
    1. Comparativa: Cuánto le costaría hacer esto con un desarrollador in-house ($200k/año) 
       vs usar el SDK de BeZhas.

    # Ejecución en Gemini CLI

Términos y Condiciones Técnicos: BeZhas SDK & Governance Protocol
1. Objeto de la Licencia y Uso del SDK
BeZhas otorga una licencia limitada, no exclusiva y revocable para utilizar el BeZhas SDK con el fin de interactuar con la API de automatización de flujos de trabajo y el protocolo de tokenización de activos (RWA).

Integración: El usuario es responsable de la implementación técnica siguiendo la documentación oficial.

Automatización: El uso del SDK para automatizar movimientos de fondos o acuñación de tokens es responsabilidad exclusiva del poseedor de las llaves privadas (Private Keys).

2. Modelos de Acceso y Facturación
El acceso a las funciones avanzadas del SDK se rige por dos modalidades:

Suscripción Mensual (SaaS Tier): Pago fijo mensual que garantiza cuotas de llamadas a la API (Rate Limits) superiores. Los pagos realizados en BEZ-Coin disfrutan de un 20% de descuento sobre la tarifa base.

Modelo de Consumo (Pay-as-you-go): Facturación basada en unidades de cómputo por transacción. El coste se deduce del balance de tokens depositado en el contrato de servicio.

3. Gobernanza y Participación Activa (BEZ-Coin)
El uso del SDK no solo otorga funcionalidad técnica, sino derecho de influencia:

Poder de Voto: Los suscriptores que mantengan un balance mínimo de BEZ-Coin podrán proponer y votar cambios en la estructura de comisiones del SDK.

ROI de Gobernanza: Una fracción de las comisiones recolectadas por el uso del SDK a nivel global se redistribuirá o se utilizará para recompras (buy-backs) según lo decida la DAO, beneficiando a los poseedores de tokens de gobernanza.

4. Garantías y SLA (Service Level Agreement)
BeZhas se compromete a una disponibilidad de la API del 99.9% para usuarios en niveles de suscripción.

Seguridad: El SDK ha sido auditado para prevenir ataques de reentrada. Sin embargo, BeZhas no se hace responsable de la pérdida de activos por una mala gestión de las llaves privadas por parte del integrador.

Actualizaciones: Las actualizaciones críticas de seguridad se desplegarán automáticamente a través del repositorio oficial.

5. Propiedad Intelectual y RWA
Al utilizar el SDK para la tokenización de activos reales (Real World Assets), el integrador garantiza que posee los derechos legales sobre dichos activos. BeZhas proporciona la infraestructura técnica, pero no actúa como custodio ni validador legal de los activos subyacentes.
